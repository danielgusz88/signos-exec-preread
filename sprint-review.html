Meal Logging (CaliBite)

Demo: 12:36 ‚Äì 50:06 ‚Ä¢ Led by David, Pierre, Maggie, Alex, Noah
New Product AI Meal Capture Recipe Import

‚úÖ What Was Built

  • Photo ‚Üí instant meal log: Snap a photo, AI identifies food, logs calories/macros instantly in background
  • Barcode & nutrition label scanning: Auto-detects barcodes, resolves via FatSecret database
  • Recipe import from Instagram/TikTok: Share any link, scraper extracts ingredients, nutrition, and instructions
  • Macro budgets: First-ever daily macro targets for Signos users (protein, fiber, etc.)
  • Quick verdict scores: Instant health rating on every logged meal
  • Voice & text logging: Multiple input channels ‚Äî "I had two eggs" works
  • Intent-based meal updates: Say "actually I had 4 eggs" and it updates the existing meal in place
  • Admin QA panel: Internal tool for testing all capture pipelines, allowing QA before mobile merge
  • Suggested items for completeness: AI suggests what might be missing from a meal ("did you also have yogurt?")
  • Glucose estimates (PPGR preview): Population-level glucose impact estimates without CGM

üí° Key Learnings

  • "Signos is for lazy people": Optimized for minimal effort ‚Äî one tap from home screen to camera, like Snapchat's mission
  • 400+ new AI-generated files: Speed of development meant very limited code review. Major AI debt to validate
  • Internal tooling is powerful: Admin panel lets QA and PMs test pipeline pieces independently before mobile integration
  • Multi-input complexity: Voice, photo, barcode, URL, text ‚Äî each has different processing pipelines and failure modes
  • FatSecret may not be the best source: Some reliability issues with nutrition data ‚Äî exploring alternatives

üéØ What's Left for V1

  • Validate and stabilize all 400+ AI-generated files
  • Wire up water logging (partially done)
  • Saved & favorite meals with food swapping
  • Weekly reports and insights
  • Restaurant menu scanning
  • Barcode/nutrition label end-to-end integration
  • Instrument metrics ‚Äî need to track what people are doing
  • Settings, reports, privacy/consent compliance
  • Better PPGR graph integration (backend exists, not in UI)
  • Edit functionality on iOS (not yet merged)

‚ö†Ô∏è Risks & Dependencies

  • Code quality concern: "A lot that was AI-generated and just the amount and the speed meant there's not been a lot of code review or cleanup"
  • Data reliability: FatSecret may not consistently provide accurate nutrition data
  • Feature branch only: Many features built in feature branches, not merged to main
  • iOS-Android parity: Different outputs from same PRD inputs
  • Free ‚Üí CGM conversion path: Not yet designed how meal logging leads to CGM upsell
